# 记忆分类目录设计
## 基于 O-COLOR 业务的自定义记忆架构

### 记忆不是"wing/hall/room"，是按你的业务来的

---

## 一级分类（6大记忆仓）

### 📦 仓1：商业战略
**涵盖**：O-COLOR定位、圆网打法、三才竞争、产业链分析、定价策略
**核心用途**：每次讨论商业决策时，自动关联之前的所有战略讨论
**存储位置**：`memory/vaults/商业战略/`
**现有文件数**：约26个文件涉及

### 📦 仓2：产品技术
**涵盖**：二代艺术漆产品参数、圆网墙布工艺、成本核算、配方、设备
**核心用途**：讨论产品改进、成本控制、工艺优化时的参考
**存储位置**：`memory/vaults/产品技术/`
**现有文件数**：约20个文件涉及

### 📦 仓3：市场渠道
**涵盖**：展会、经销商、加盟商、师傅生态、小程序裂变、竞品信息
**核心用途**：讨论渠道策略、市场开发时调用
**存储位置**：`memory/vaults/市场渠道/`
**现有文件数**：约15个文件涉及

### 📦 仓4：经验与学习
**涵盖**：毛选、资治通鉴、盐铁论、左传、东周列国志、遥远的救世主、行业调研、竞争分析
**核心用途**：从书和实战中提炼的智慧，变成商业决策的依据
**存储位置**：`memory/vaults/经验学习/`
**现有文件数**：约47个文件涉及

### 📦 仓5：系统技术
**涵盖**：OpenClaw、MemFlow AI、记忆系统、网站维护
**核心用途**：技术决策的追溯、系统维护记录
**存储位置**：`memory/vaults/系统技术/`
**现有文件数**：62个全部文件涉及

### 📦 仓6：财务运营
**涵盖**：成本核算表、库存统计、营收日报、费用记录
**核心用途**：经营数据分析、财务决策
**存储位置**：`memory/vaults/财务运营/`
**现有文件数**：约17个文件涉及

### 📦 仓7：软件制作
**涵盖**：成本核算表程序、库存统计系统、小程序、网页工具、脚本开发
**核心用途**：所有自己写的软件工具，方便复用和升级
**存储位置**：`memory/vaults/软件制作/`
**现有文件数**：约61个文件涉及（覆盖面最广，含各类工具脚本）

---

## 分类的逻辑

不是按"时间"也不是按"技术格式"，是**按你的业务动作**：

- **做战略决策时** → 查仓1（商业战略）和仓3（市场渠道）
- **做产品时** → 查仓2（产品技术）
- **学东西想用上时** → 查仓4（书单学习）
- **搞技术时** → 查仓5（系统技术）
- **算账时** → 查仓6（财务运营）

## 落地方式

不是把现有文件搬过去，而是：
1. 在语义索引上加**标签字段**（每个记忆片段标记属于哪个仓）
2. 搜索时可以先限定仓再搜，更精准
3. 每次对话后自动记笔记时，自动打上仓标签
